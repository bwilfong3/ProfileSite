  • The Solar Army

    Tools and languages used:

    The Solar Army project at Fairmont State University aims to find a photoelectrocatalyst that can aid in splitting water into hydrogen that can be used for fuel.

    Over summer in 2016, Dr. Erica Harvey of the chemistry department at FSU asked me if I could help them organize their data and possibly create some visualizations on the web. At the time, their data was only being stored in Excel Spreadsheets. There was a LOT of work to do!

    I love learning. This project really gave me an opportunity to get my hands dirty with a bunch of technologies I had not used before. Here is a list of some of the things I've done:

    • Created a MySQL database to house the data being stored in excel spreadsheets
    • Created a website using Drupal 7.
    • Used PHP, HTML5 forms, and SheetJS to create a page that can parse our data files and then store the extracted information in the database
    • Created an existence matrix using D3 that shows which elements have been tried together and how many times. The matrix has dynamic tooltips that show the counts and elements. All of the squares are also clickable and will show a histogram of the combination's use with other elements

    Link to website: http://solararmy.fsuhosting.com

    Fun stuff!: http://solararmy.fsuhosting.com/data/matrix_visualization.html

  • Credit Bro

    Tools and languages used:

    Credit Bro is a personal project that I've been working on. Most credit card repayment calculators on the web give you the time and interest paid for a single card. I am trying to create a tool that, given a monthly budget, payments are allocated in such a way to minimize interest paid.

    Features:

    • Users can add as many credit cards as needed.
    • A credit card consists of a balance, APR, minimum payment, and optional name
    • The user can select different payment algorithms (highest balance first, highest interest first, etc.)
    • The website will compute the total interest and show a detailed payment plan for each card.
    • Credit cards are stored as a cookie so users need not re-enter the information after leaving the site.

    Future Goals:

    • I would like to have an algorithm that will compute minimum interest possible. I've been spitballing ideas, but I have yet to come up with something that is less than 2^n complexity, where n is the number of cards. I may still implement this algorithm, but limit the number of cards as to not crash the browser.
    • Users can print their detailed payment schedule in a clean format.

    Link to GitHub repo: https://github.com/bwilfong3/CreditBro
